APM Login

Let's start with the basic application flows to identify how are the applications interconnected
Before you can start developing and testing your app against the sandbox you have to login into the Appication Manager.
APM allows 3rd party developers to manage the access to the fidor API and build and test their Apps against the sandbox.

Create an App

From the your apps list page you can create a new app
By doing so, you have to provide some mandatory information that will be used afterwards to for example, prevent
malicious apps to access your API account. First of all, whenever you want to acquire a new API token for yourself
of for on of the users who use your app, you have to provide us with you client_id and client_secret.
This way we will know that it's your app that wants to get access. Additionally to that we will check if the
request is coming from one of the allowed_ip addresses and if the user will be redirected to the
callback_url you provided. Whenever you create a new App a new sandbox is created and and associated with it.

Sandbox Page

on the application level).

Architectural flavors
HTTP/2 leaves most of HTTP 1.1's high-level syntax, such as methods, status codes, header fields, and URIs, the same. The element that is modified is how the data is framed and transported between the client and the server. HTTP/2 - minification, server push
SOAP - Simple Object Access Protocol. It uses XML Information Set for its message format,
and relies on application layer protocols, most often Hypertext Transfer Protocol (HTTP)
or Simple Mail Transfer Protocol (SMTP), for message negotiation and transmission.
REST - Representational state transfer
WebRTC - (Web Real-Time Communications)
Websockets
There is one significant difference: Websockets works via TCP, WebRTC works via UDP. In fact, WebRTC is SRTP protocol with some addutional features like STUN, ICE, DTLS etc and internal VoIP features such as Adaptive Jitter Buffer, AEC, AGC etc.

DDD = Domain-driven design is an approach to developing software for complex needs by deeply connecting the implementation to an evolving model of the core business concepts.
Event Sourcing = (Immutable Data) The fundamental idea of Event Sourcing is that of ensuring every change to the state of an application is captured in an event object, and that these event objects are themselves stored in the sequence they were applied for the same lifetime as the application state itself.
Why is it important for the banks - cause it's how the banks should work. They should be able to track back any
change in the system, because they obliged to do so by law.
CQRS - command query responsibility segregation. It solves the problem of building the reporting queries for example
cause with properly built event sourced system it's quite hard to generate them. Additionally to that
it helps to segregate the concerns of writing to the system and reading from it.
